## PRIMARY PROJECT: RX MANAGER (Prescription Management App)

### Project Overview
**What It Is:** Mobile app for medication management designed for disabled users with complex prescription schedules
**Built For:** Real user (girlfriend with kidney transplant + stroke recovery)
**Problem Solved:** Medication chaos, missed doses, pharmacy coordination across multiple providers

### Technical Stack
- **Frontend:** React Native (cross-device compatibility)
- **Backend:** Supabase (real-time sync)
- **Local Storage:** SQLite + AsyncStorage (offline-first)
- **Security:** Encrypted patient data, HIPAA-aware design
- **Intelligence:** MCP mathematical bridge, pattern learning system

## UX INNOVATIONS (Portfolio-Worthy Details)

### 1. Medical Personalization Engine
**Inspired by:** Figma Personal Style DNA system (cross-domain research application)
**How It Works:**
- Complexity slider 0-100% (like Figma's style intensity)
- Auto-adjusts UI based on user comfort level
- Low complexity = large buttons, high contrast, essentials only
- High complexity = compact controls, comprehensive info

**Why It Matters:** Not one-size-fits-all. Adapts to cognitive load and medical literacy.

### 2. Device Tier Adaptation
**3 Performance Tiers:**
- **Tier 1 (Legacy):** Simple text lists, basic notifications, minimal animations
- **Tier 2 (Standard):** Rich cards, medication photos, progress indicators
- **Tier 3 (Enhanced):** AI insights, pattern recognition, predictive suggestions

**Why It Matters:** Works on ANY device, from old Android to latest iPhone.

### 3. Thermal-Responsive Layout
**Unique Feature:** UI simplifies when device overheats
**How:** Caches data, reduces animations, prioritizes critical functions
**Why It Matters:** Real-world consideration for users with older/budget devices.

### 4. Visual Weight System
**Hierarchy Based on Medical Priority:**
- Emergency protocols = Weight 10 (always visible, largest)
- Medication names = Weight 8 (primary focus)
- Schedule times = Weight 6 (important but supporting)
- Notes/instructions = Weight 3 (supplementary)

**Why It Matters:** Safety-first design. Life-critical info can't be buried.

### 5. Progressive Disclosure
**Problem:** Too much info overwhelms disabled/ADHD users
**Solution:** Feature toggles reveal complexity as needed
**Example:**
- Basic mode: "Take 2 pills now"
- Advanced mode: Shows dosage, interactions, refill status, insurance info

### 6. Conversational Setup
**Inspired by:** Figma's natural language template system
**How It Works:**
- User: "I take metformin twice daily with breakfast"
- App: Auto-generates schedule, sets reminders, links to pharmacy

**Why It Matters:** Removes technical barrier. No forms, just conversation.

### 7. Caregiver Access System
**Role-Based Permissions:**
- Owner: Full control
- Primary caregiver: View schedule, get alerts, emergency access
- Extended family: View-only, limited notifications

**Why It Matters:** Real coordination between patient and support network.

## DESIGN PROCESS HIGHLIGHTS

### Phase 1: User Research (Real-World Problem)
- Girlfriend's actual medication chaos
- 8+ prescriptions across 3 pharmacies
- Insurance coordination nightmares
- Missed doses = health risks

### Phase 2: Competitive Analysis
**Studied:**
- AdlerRx (accessibility leader)
- ClearRx (visual hierarchy)
- Standard pharmacy apps (identified failures)

**Key Insight:** Existing apps assume medical literacy and stable device access. Built for healthy people managing meds, not disabled users.

### Phase 3: Cross-Domain Research Application
**Innovation:** Applied Figma plugin research to medical UX
- Personal Style DNA → Medical Personalization Engine
- Node-based design → Medical decision trees
- Natural language templates → Conversational setup

**Why This Matters for Portfolio:** Shows ability to transfer knowledge across domains.

### Phase 4: Iterative Prototyping
**Tools:** Figma (wireframes) → React Native (functional prototype) → Real user testing
**Iterations:** 15+ UI revisions based on girlfriend's feedback
**Result:** App she actually uses daily

### Phase 5: Accessibility-First Design
**WCAG 2.1 AA Compliance:**
- High contrast modes
- Screen reader optimization
- Voice input support
- Large touch targets (44x44px minimum)
- Color-blind friendly palettes

## TECHNICAL UX DECISIONS

### Offline-First Architecture
**Why:** Can't assume reliable internet (rural areas, budget plans)
**How:** Local SQLite database syncs when connection available
**UX Impact:** App works even during pharmacy dead zones

### Pattern Learning System
**What It Does:** Learns user habits over time
- Notices "always takes morning meds late on weekends"
- Adjusts reminder times automatically
- Predicts refill needs based on adherence patterns

**Privacy:** All learning happens locally, never sent to servers

### Multi-Modal Input
**Why:** Accessibility isn't one thing
- Touch: Standard interface
- Voice: "Add medication" commands
- Text: For complex instructions

**UX Impact:** Works for users with limited dexterity, vision, or cognitive differences
